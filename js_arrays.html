JavaScript Arrays

JavsScript 배열은 단일 변수에 여러 값을 저장하는 데 사용됩니다.

ex

var cars = [ "Saab", "Volvo", "BMW"];
직접해보기

배열이란 무엇입니까?

배열은 한번에 둘 이상의 값을 저장할 수 있는 특수 변수 입니다.

항목 목록 (예 : 자동차 이름 목록)이 있는 경우 단일 변수에 자동차를 저장하면 다음과 같이 될수 있습니다.

var car1 = "Saab";
var car2 = "Volvo";
var car3 = "BMW";

그러나 자동차를 돌아보고 특정 차량을 찾으려면 어떻게해야할까요? 3대가 아니라 300대가 있다면?
해결책은 배열입니다!
배열은 하나의 이름으로 여러 값을 가질 수 있으며 인덱스 번호를 참조하여 값에 액세스 할 수 있습니다.

배열 요소가 객체가 될 수 있음

JavaScript 변수는 객체가 될 수 있습니다. 배열은 특별한 종류의 객체입니다.

이 때문에 동일한 배열에서 여러 유형의 변수를 가질 수 있습니다.

배열에 객체를 가질 수 있습니다. 배열에 함수를 포함 할 수 있습니다. 배열에 배열을 가질수 있습니다.

배열 속성 및 메서드

JavaScript 배열의 진정한 강점은 기본 제공 배열 속성 및 메서드 입니다.

예제들

var x = cars.length; // The length property
returns the number of elements
var y = cars.sort();

배열 메서드는 다음 장에서 다룹니다.

길이 속성

배열의 length속성은 배열의 길이 (배열 요소의 수)를 반환합니다.

예

var fruits = ["Banana","Orange","Apple","Mango"];
fruits.lenth;// the lenth of fruits is 4
직접해보기
length 속성은 항상 가장 높은 배열 인덱스보다 하나 더 큽니다.

루핑 배열 요소

배열을 반복하는 가장 좋은 방법은 "for"루프를 사용하는 것입니다.

예

var fruits, text, flen, i;

fruits = ["Banana", "Orange", "Apple", "Mango"];
flen = fruits.lenth;
text ="<ul>";
for ( i = 0; i < flen; i++){
    text += "<li>" + fruits[i] + ""</li>";
}

연관 배열

많은 프로그래밍 언어는 명명 된 인덱스를 사용하여 배열을 지원합니다.
명명된 인덱스가 있는 배열을 연관 배열 (또는 해시)이라고 합니다.
JavaScript는 명명 된 인덱스가 있는 배열을 지원 하지 않습니다. 자바 스크립트에서 배열은 항상 번호가 매겨진 색인을 사용 합니다.

예

var person = [];
person[0] = "John";
person[1] = "Doe";
person[2] = "46";
var x = person.length; // person.length will return 3
var y = person[0]; // person[0] will return "John"
}
직접해보기
경고!!
명명된 인덱스를 사용하면 JavaScript가 배열을 표준 객체로 다시 정의합니다.
그런 다음 일부 배열 메서드와 속성은 잘못된 결과를 생성 합니다.

예

var person = [];
person["firstName"] = "John";
person["lastName"] = "Doe";
person["age"] = "46";
var x = person.length; // person.length will return 0
var y = person[0]; // person[0] will return undefined
}

배열과 객체의 차이점

자바 스크립트에서 배열은 번호가 매겨진 색인을 사용 합니다.
자바 스크립트에서 객체 는 명명 된 색인을 사용합니다.

배열은 번호가 매겨진 인덱스가 있는 특별한 종류의 객체입니다.

배열을 사용해야 하는 경우. 객체 사용시기

  • 자바 스크립트는 연관 배열을 지원하지 않습니다.
  • 요소 이름을 문자열 (텍스트)로 만들려면 객체를 사용해야 합니다.
  • 당신은 사용해야 배열을 당신이 할 요소 이름을 할때 번호.

새 Array()를 피하십시오.

JavaScript의 기본 제공 배열 생성자 new Array()를 사용할 필요가 없습니다.
대신 [] 을 사용하십시오.
이 두 개의 다른 명령문은 모두 points라는 새 빈 배열을 만듭니다.

var points = new Array(); // Bad
var points = []; // Good
}

이 두 개의 서로 다른 명령문은 모두 6개의 숫자를 포함하는 새로운 배열을 만듭니다.

예

var points = new Array(40, 100, 1, 5, 25, 10); //Bad
var points = [40, 100, 1, 5, 25, 10] //Good
}

새로운 키워드는 코드를 복잡하게 한다. 또한 예기치 않은 결과가 발생할 수 있습니다.

var points = new Array(40, 100); // Creates an array with two elements ( 40 and 100)
}

요소 중 하나를 제거하면 어떻게 될까요?

배열을 인식하는 방법

일반적인 질문은 변수가 배열인지 어떻게 알 수 있을까?
문제는 JavaScript 연산자 typeof가 "object"를 반환 한다는 것입니다.

var fruits = ["Banana", "Orange", "Apple", "Mango"];
typeof fruits; //returns object }
직접해보기

typeof 연산자는 JavaScript 배열이 객체이기 때문에 object를 반환합니다.

해결책 1

이 문제를 해결하기 위해 ECMAScript 5에서는 Array.isArray()라는 새 메서드를 정의합니다.

Array.isArray(fruits); //returns true }
직접해보기

이 솔루션의 문제점은 이전 브라우저에서 ECMAScript5가 지원되지 않는다는 것 입니다.

해결책 2

이 문제를 해결하기 위해 자신 만의 isArray()함수를 만들 수 있습니다:

function isArray(x) {
return x.constructor.toString().indexOf("Array") > -1; }
직접해보기

인수가 배열이면 위의 함수는 항상 true를 반환합니다.
또는 더 정확하게 : 객체 프로토타임에 "Array"라는 단어가 포함되어 있으면 true를 반환합니다.

해결책 3 :

의 instanceof객체가 주어진 생성자에 의해 생성되는 경우는 true:

var fruits = ["Banana", "Orange", "Apple", "Mango"]; fruits instanceof Array //returns true }
직접해보기
